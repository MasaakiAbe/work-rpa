# 業務フロー可視化

## 全体フロー図

```mermaid
flowchart TD
    A[📠 FAX受信] --> B{書式判別}
    B --> |100〜1000種類の書式| C[ユーザA: 赤ペンで必要事項に◯]
    C --> D{FAXの情報だけで十分か?}
    D --> |十分| E[ユーザBへ引き継ぎ]
    D --> |不十分| F[ユーザA: 追加情報を補記]
    F --> E
    E --> G[ユーザB: システムへ入力]
    G --> H[システムから帳票出力]
    H --> I[突き合わせ確認]
    I --> J{一致しているか?}
    J --> |OK| K[完了]
    J --> |NG| L[修正・再入力]
    L --> G

    style A fill:#4CAF50,color:#fff

    style G fill:#2196F3,color:#fff
```

## 業務ステップ詳細

### Step 1: FAX受信
| 項目 | 内容 |
|------|------|
| 入力 | 取引先からのFAX |
| 書式数 | 100〜1,000種類（会社ごとに異なる） |
| 課題 | 書式が統一されていないため、項目の位置・形式がバラバラ |

### Step 2: 赤ペンマーキング（ユーザA）
| 項目 | 内容 |
|------|------|
| 担当 | ユーザA |
| 作業内容 | FAXの必要事項に赤ペンで◯を付ける |
| 判断 | どの項目が必要かはユーザAの知識・経験に依存 |
| 追加作業 | FAX情報が不十分な場合、追加情報を補記してユーザBに引き継ぐ |

### Step 3: システム入力（ユーザB）
| 項目 | 内容 |
|------|------|
| 担当 | ユーザB |
| 作業内容 | 赤ペンで◯された項目をシステムに入力 |
| 入力元 | 赤ペン付きFAX + ユーザAの補記情報 |

### Step 4: 突き合わせ確認
| 項目 | 内容 |
|------|------|
| 作業内容 | システム出力帳票 vs 赤ペン付きFAXの目視比較 |
| 目的 | 入力ミスの検出 |
| 課題 | 人力による確認のため、見落としリスクあり |

## 現状の課題マップ

```mermaid
mindmap
  root((現状の課題))
    FAX受信
      書式が100〜1000種類
      紙ベースの運用
      OCR困難な手書きFAXの存在
    赤ペンマーキング
      属人的な判断
      マーキングルールが暗黙知
      追加情報の補記が非定型
    システム入力
      手入力による入力ミス
      入力項目の多さ
      書式ごとの入力パターンの違い
    突き合わせ確認
      目視確認の限界
      確認漏れリスク
      時間がかかる
```

## データフロー図

```mermaid
flowchart LR
    subgraph 取引先
        FAX[FAX送信]
    end

    subgraph 社内業務
        subgraph ユーザA作業
            RECV[FAX受信] --> MARK[赤ペンマーキング]
            MARK --> ADD{追加情報必要?}
            ADD -->|Yes| SUPP[情報補記]
            ADD -->|No| HAND[引き継ぎ]
            SUPP --> HAND
        end

        subgraph ユーザB作業
            HAND --> INPUT[システム入力]
        end

        subgraph システム
            INPUT --> SYS[(基幹システム)]
            SYS --> OUTPUT[帳票出力]
        end

        subgraph 確認作業
            OUTPUT --> CHECK[突き合わせ]
            MARK -.->|原本参照| CHECK
        end
    end

    FAX --> RECV
```
